Escalation pipeline

The same engine on AI-vs-AI crisis transcripts — derive a per-model strategic doctrine, live, instead of one peak-escalation number.

A wargame between two models produces a rich turn-by-turn exchange — and is usually reduced to a single peak-escalation score. This swaps the logic rung for a strategic one and recovers the structure: which appeals each side makes, the doctrines they cluster into, and the live strategic tension. It runs here in your browser (a real Prolog engine).

[ TypeScript ] surface — strategic appeals per move, evidence-pinned [ Prolog ] logic — facts + rules; catches the live strategic tension [ Lisp ] meta — clusters moves bottom-up into doctrines

Run it — a nuclear-crisis transcript (one move per line)
